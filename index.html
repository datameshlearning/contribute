Contribute to the Data Mesh Community

If you are interested in contributing to the overall data mesh community and/or Data Mesh Learning, welcome! The easiest/best way to contribute is probably just being active in the Slack community, connecting people to resources and being helpful. There are many other ways you could contribute so if you have an idea, we would love to hear it, email us or message Scott Hirleman in the Slack community. That said, there are some things we've come up with below:

Community promotion: generally, just telling people about the community. If you find value from it and want others to experience the same, tell them! :D The easiest way to sign up is to send them here: https://launchpass.com/data-mesh-learning

Content review (prior to publishing): we want to offer people creating content (user stories, blog posts, presentations, videos, etc.) a chance for others to review and give feedback before publishing. If you want to volunteer to review content before it gets published, please sign up using this Google Form. Contact information provided will only be used for the purpose of content review.

Content classification: we are attempting to build a repository of data mesh and data mesh tangential content. Thus far, it is quite, quite early days... Not experts on data classification and management over here so some friendly guidance on how to handle would be great. The plan is for an information resource AND a relatively easy way to find good links to put into newly created content. Links like that are good for informational purposes but also help drive up SEO and trust rankings.

Slack community management: Once we've seen you in action in the Slack channel, we are happy to share the load of managing the data mesh community. We don't believe communities should be controlled by a single person and we want to be able to bring many people in who share the same goals of collaboration and communication and a committment to be excellent to each other.

Website management: this "website" in it's current form is a bit of a hot mess, based off a single great tutorial on GitHub Pages. No really, it was great, it just isn't really for what this morphed into. So, as the community picks up steam, it is likely to get moved to a formal website with a WordPress system and everything. If you want to be part of that, it would be greatly, greatly appreciated ;)

Other ideas? as we said, let us know in Slack or via email.
